Comparison of Algorithms

Jarvis March Kirkpatrick-Seidel
Algorithmic Approach Straightforward, iterative method: selects the next point on the convex hull by considering all points and choosing the one with the smallest angle. More sophisticated algorithm which uses divide-and-conquer: recursively partitions the points and merges the convex hull of these subsets.
Time Complexity O(nh), where n is the number of input points and h is the number of points on the convex hull. Runs in O(n^2) time in the worst case. O(n log h), where n is the number of input points and h is the number of points on the convex hull. Better than Jarvis March, especially for large datasets.
Applicability Suitable for small to medium sized datasets, used when simplicity is preferred over efficiency. Suitable for large datasets with larger number of points, offers better performance due to superior time complexity.
Robustness Can handle degenerate cases (such as collinear points) but may suffer from inefficiency in such scenarios. More robust in handling degenerate cases due to its sophisticated merging technique.
